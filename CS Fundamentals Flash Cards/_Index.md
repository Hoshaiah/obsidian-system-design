# Flashcards — Computer Science Fundamentals

The high-yield CS knowledge for technical interviews and engineering work. Each note is its own subdeck (`flashcards/<topic>`) for the Obsidian *Spaced Repetition* plugin.

## Decks
- [[Data Structures]] — arrays through B-trees, what to pick when
- [[Algorithms & Complexity]] — Big-O, sorting, DP, greedy, graph search
- [[Operating Systems]] — processes, virtual memory, scheduling, syscalls
- [[Networking]] — TCP/UDP, TLS, HTTP versions, DNS, NAT
- [[Databases (CS)]] — ACID, indexes, isolation, MVCC, SQL vs NoSQL
- [[Concurrency & Threading]] — races, locks, memory models, async vs threads
- [[Memory Management]] — stack/heap, GC, RAII, cache lines, leaks
- [[Computer Architecture]] — pipelines, caches, branch prediction, NUMA
- [[Programming Paradigms]] — OOP, FP, SOLID, composition over inheritance
- [[Security Fundamentals]] — authn/authz, TLS, OWASP, JWT, OAuth

## How to use
- Pareto rule: these are the most reused ideas. Master cold before adding breadth.
- Add your own cards from misses — those are worth 10x pre-made ones.
- Cards use the `Question`/`?`/`Answer` format used by Obsidian Spaced Repetition.
